# 项目改进计划 (Project Improvement Plan)

根据《开题报告》的要求与当前系统实现情况的对比分析，本计划旨在补全功能缺失、优化用户体验，并确保系统满足毕业设计的学术与工程标准。

## 1. 现状分析 (Status Analysis)

| 核心要求 (Requirements) | 当前状态 (Current Status) | 差距与问题 (Gaps) |
| :--- | :--- | :--- |
| **用户与权限管理** | 已实现基础登录/注册 | 缺少系统操作日志 (System Logs)；缺少用户资料修改与删除功能。 |
| **流程模型管理** | 已实现创建/列表 | 缺少编辑 (Edit) 和删除 (Delete) 功能；缺少版本管理 (Version Control)。 |
| **数据采集** | 基础时间节点记录 | 数据维度单一，仅记录了时间，未记录具体的业务表单数据。 |
| **可视化监控** | 已实现基础看板 | 仪表盘与预测分析页面混淆；缺少更深度的图表分析。 |
| **算法预测** | 基础随机森林实现 | 缺少算法对比实验 (Benchmark)；缺少影响因子分析功能。 |
| **前端体验** | 已进行初步美化 | 登录页已优化；按钮风格已统一；但部分交互细节仍需打磨。 |

## 2. 改进路线图 (Roadmap)

### 第一阶段：前端体验与基础架构优化 (Phase 1: UI/UX & Architecture) - **[进行中]**
*   [x] **登录页面重构**: 增加动态背景与玻璃拟态效果，提升第一印象。
*   [x] **仪表盘与分析页分离**:
    *   `Dashboard`: 专注实时运行状态（活跃实例、资源负载）。
    *   `Analytics`: 专注历史数据分析、算法准确率趋势、误差分布。
*   [x] **全局样式统一**: 统一按钮 (`el-button`)、卡片 (`el-card`) 和圆角设计，确保风格一致性。
*   [ ] **路由结构优化**: 确保菜单跳转逻辑清晰。

### 第二阶段：核心功能补全 (Phase 2: Core Features) - **[本周计划]**
*   **系统日志 (System Logs)**:
    *   *后端*: 新建 `SystemLog` 表，记录 `user_id`, `action`, `timestamp`, `details`。
    *   *前端*: 在“系统维护”或“用户管理”下增加日志查看页面。
*   **流程模板增强**:
    *   实现 **编辑** 功能：允许修改流程名称和节点结构。
    *   实现 **版本控制**：修改模板时，保留旧版本，生成新版本号 (v1.0 -> v1.1)。
    *   实现 **删除** 功能（软删除）。

### 第三阶段：算法与数据深度挖掘 (Phase 3: Algorithm & Data)
*   **算法对比实验**:
    *   在后端集成 `LinearRegression` 和 `SVM` 作为基准模型。
    *   在 `Analytics` 页面展示不同模型的 MSE/R2 对比图表，满足“对比实验”的学术要求。
*   **多维数据采集**:
    *   扩展 `NodeExecution` 表，增加 `input_data` (JSON) 字段，模拟采集业务数据（如金额、数量）。

### 第四阶段：系统测试与论文支撑 (Phase 4: Testing & Thesis)
*   **压力测试**: 使用脚本模拟 100+ 并发流程实例，验证系统稳定性。
*   **导出功能**: 支持导出运行历史数据 (CSV/Excel)，用于论文数据分析。

## 3. 下一步具体行动 (Next Actions)

1.  **后端**: 定义 `SystemLog` 模型并编写 CRUD 接口。
2.  **后端**: 完善 `WorkflowTemplate` 的 `PUT` (更新) 和 `DELETE` 接口，加入版本号逻辑。
3.  **前端**: 在 `Analytics.vue` 中对接真实的后端预测数据接口（目前为 Mock 数据）。

---
**备注**: 此计划将确保您的毕业设计不仅在功能上满足任务书要求，而且在界面和技术深度上达到优秀水平。
